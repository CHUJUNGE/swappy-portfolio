Project Iterations

Continuous improvement process based on user feedback

Iteration Methodology

Design Thinking Iteration Cycle

We adopted the design thinking methodology for project iterations, including the following steps:

  1. Empathize: Deeply understand user needs through interviews and observations
  2. Define: Clearly identify user pain points and problems to be solved
  3. Ideate: Brainstorm multiple possible solutions
  4. Prototype: Transform selected solutions into testable prototypes
  5. Test: Have users test the prototype and collect feedback

Each iteration follows this cycle, ensuring the product continuously improves and better meets user needs.

Main Iteration Journey

Iteration 1: Initial Prototype Testing

User Feedback

  • Navigation structure not clear enough, users had difficulty finding specific features
  • Item upload process too complicated with too many steps
  • Token system concept not easy to understand, needed clearer explanation
  • Item categories not detailed enough, difficult to filter precisely

Improvements Made

  • Redesigned navigation bar, added dropdown menus and icon indicators
  • Simplified item upload process, reducing steps from 7 to 4
  • Added token system tutorial and visual explanations
  • Expanded item categorization system, adding secondary and tertiary categories

Iteration 2: User Interface Optimization

User Feedback

  • Poor mobile adaptation, some elements displayed abnormally
  • Insufficient color contrast affecting readability
  • Lack of visual feedback, users uncertain if operations were successful
  • Image upload functionality frequently failed

Improvements Made

  • Comprehensively optimized mobile responsive design, ensuring good experience across various screen sizes
  • Improved text-to-background contrast ratio, meeting WCAG AA standards
  • Added animations and toast notifications to enhance operation feedback
  • Refactored image upload functionality, adding progress bars and error handling

Iteration 3: Feature Expansion

User Feedback

  • Users wanted more social features, such as comments and private messaging
  • Need for more convenient item search methods
  • Desire to see transaction history and statistical data
  • Some users reported insufficient accessibility features

Improvements Made

  • Added item comment system and private messaging between users
  • Implemented advanced search and filtering functionality, supporting multiple criteria combinations
  • Developed personal center page, displaying transaction history and environmental contribution statistics
  • Comprehensively enhanced accessibility support, including screen reader compatibility and keyboard navigation

User Testing Methods

Task Testing

Have users complete specific tasks such as uploading items, searching for items, and requesting exchanges, while observing completion rates and difficulties encountered.

Usability Observation

Through eye tracking and behavior recording, analyze how users interact with the interface, identifying points of confusion and opportunities for optimization.

Feedback Interviews

Conduct in-depth interviews with users after testing to understand their experiences, feelings, and suggestions, collecting qualitative feedback.

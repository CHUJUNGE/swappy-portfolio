Physical Store Prototype

Design Objectives

A miniature, desktop-scale prototype was created to simulate the in-store swapping experience, focusing on spatial accessibility, clear signage, and user flow optimization.

  • Simulate real-world spatial experience with a store layout.
  • Test navigation flow and aisle accessibility for users with mobility or vision impairments.
  • Communicate how physical and digital components connect during the user journey.

Physical
Experience

Prototype Construction

The prototype was built using dolls, foam boards, and cardboard. Visual icons, directional arrows, and color-coded zones were added to simulate signage, item categories, and navigation guides.

What We Tested

  • Navigation flow clarity for students with vision and mobility needs
  • Layout effectiveness for swap interactions
  • Integration of tactile, visual, and spatial design elements

Prototype Iterations

We developed two versions of our physical prototype to test and refine the in-store user experience. Iterative design helped us address layout clarity, accessibility barriers, and operational realism.

Version 1: Circular Event-Based Layout

Purpose: Simulate a temporary swap event experience with a circular booth structure.

Initial event-based circular layout built with Lego.

Users enter from one side and walk around in a loop.
Items are displayed in stations arranged in a circular pattern.
Minimal signage, open exploration.
Feedback was collected during in-class walkthrough.
Issues Identified:
  • Confusing navigation flow.
  • Not wheelchair-friendly.
  • Hard to predict where items are located.
  • Volunteers had no defined roles in user support.

Version 2: Linear Store-Based Layout

Purpose: Represent a permanent, weekly-themed swap store with clear signage and wide, accessible paths.

Updated linear store layout with improved accessibility.

Straight aisle design for easier navigation.
Zoned item categories (clothing, books, accessories).
Wide aisle space (1.8m) with tactile floor guides.
Defined entry, rest, and check-out areas with volunteers on-site.

"Participants said the circular layout looked visually engaging but made it hard to plan their path—especially for those with mobility challenges."

Version Comparison

Feature Version 1: Circular Layout Version 2: Linear Store Layout
Type Temporary event Permanent store
Navigation Open loop, unstructured One-directional, tactile guided
Accessibility Minimal, hard for wheelchair users Wide paths, Braille tags, rest areas
Volunteer Role Undefined Stationed at entry, rest, and checkout
Feedback Impact Highlighted confusion and lack of flow Informed clear spatial and user zoning

Lessons from Iteration

Prototyping at physical scale helped us visualize user movement and spot accessibility issues early.

Feedback led to a major shift in how we approached spatial clarity, support roles, and item visibility.

The new design laid the foundation for both the customer journey and our interactive web prototype.

In-Store Chatbot Prototype

In-Store Support Solution

To support users inside the Swappy store — especially those with accessibility needs or unfamiliarity with the swap system — we designed a conversational chatbot to run on an in-store tablet. It guides users through item location, swap steps, and support options.

Key Features

  • Voice and text interaction
  • Visual and tactile guidance
  • Multilingual potential
  • 10–12 inch tablet display

Phase 1: Quick Prototype (Conversation Flow Design)

Before building the chatbot, we used Miro to create a high-level conversation flow to map out typical user journeys. This flow helped us clarify intent categories, accessibility options, and logical paths based on student needs.

Scenario Paths Include:

  • "I'm here to swap" vs "I'm just browsing"
  • How many items the user has
  • Finding an item by category
  • Asking for a floor map or volunteer assistance

This step helped validate the logical structure before any code or platform build-out.

Conversation flow map showing user intents and branching logic

Phase 2: Final Prototype (Interactive Web Version)

Using Voiceflow to define intents and conversation branches, we then implemented a working chatbot prototype and deployed it via GitHub Pages.
